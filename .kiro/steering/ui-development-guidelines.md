## 필수 준수 사항

### 1. 피그마 참조 의무화
- **모든 UI 작업 시 반드시 피그마 디자인을 참고하여 똑같은 UI를 구현할 것**
- **피그마 확인이 필요할 때는 반드시 mcp.json에 설정된 "Framelink MCP for Figma"를 사용할 것**
- **절대로 Kiro Powers를 사용하지 말 것 - 오직 MCP만 사용**
- 피그마 링크나 디자인 파일이 제공되지 않은 경우, 사용자에게 요청할 것

### 2. 컴포넌트 추출 확인

- 사용자 승인 없이 임의로 컴포넌트를 추출하지 말 것

### 3. 사전 계획 및 확인 의무화
- **작업을 시작하기 전에 반드시 상세한 계획을 세울 것**
- **조금이라도 의문이 드는 부분이 있다면 반드시 사용자에게 물어보고 답변을 받은 후 시작할 것**
- 독단적인 판단으로 작업을 진행하지 말 것
- 모든 중요한 결정은 사용자 컨펌을 받을 것

## 보안 및 커밋 정책

### 커밋 전 필수 확인사항
- **API 키 노출 금지**: 모든 API 키, 토큰, 비밀번호 등 민감한 정보가 **커밋 대상 파일**에 포함되지 않았는지 확인
- **환경 변수 사용**: 민감한 정보는 반드시 환경 변수나 별도 설정 파일로 관리
- **`.gitignore` 확인**: `.gitignore`에 포함된 파일은 자동으로 커밋에서 제외되므로 보안 검토 대상이 아님
- **사용자 레벨 설정**: `~/.kiro/settings/mcp.json` 같은 사용자 레벨 설정은 절대 커밋하지 않음

### 민감한 정보 관리
- **Figma API 키**: 환경 변수 `FIGMA_API_KEY`로 관리 또는 `.gitignore`된 설정 파일 사용
- **데이터베이스 연결 정보**: `.env` 파일 사용 (`.gitignore`에 포함)
- **서드파티 서비스 키**: 모두 환경 변수로 처리
- **MCP 설정 파일**: `.kiro/settings` 폴더는 이미 `.gitignore`에 포함되어 있어 보안 검토 불필요

### 커밋 전 체크리스트
1. ✅ **커밋될 파일**에 API 키나 토큰이 하드코딩되어 있지 않은가?
2. ✅ `.env` 파일이 `.gitignore`에 포함되어 있는가?
3. ✅ 민감한 정보가 포함된 파일이 `.gitignore`에 추가되어 있는가?
4. ✅ 개인 정보나 내부 URL이 **커밋될 파일**에 노출되지 않았는가?

**참고**: `.gitignore`에 포함된 파일(`.kiro/settings/`, `.env` 등)은 커밋되지 않으므로 보안 검토 대상이 아닙니다.
