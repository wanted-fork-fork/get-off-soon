# 요구사항 문서

## 소개

"곧 내려요"는 현재 앉아있으면서 곧 내릴 예정인 지하철 승객과 좌석을 찾고 있는 승객을 연결하는 양면 플랫폼 모바일 애플리케이션입니다. 이 서비스는 누가 곧 내리는지 모르는 불편함을 해결하고, 사용자 간 실시간 정보 공유를 통해 빠른 좌석 확보를 가능하게 하는 것을 목표로 합니다.

## 용어 정의

- **앉아있는_사용자**: 현재 지하철 좌석에 앉아 있으며 다음 역에서 내릴 예정인 사용자
- **서있는_사용자**: 지하철에서 사용 가능한 좌석을 찾고 있는 사용자
- **지하철_시스템**: 사용자 상호작용과 정보 공유를 관리하는 모바일 애플리케이션
- **열차_선택**: 사용자가 탑승한 특정 지하철 열차를 식별하고 선택하는 과정
- **기여도_점수**: 좌석 공유 생태계 참여에 대해 사용자에게 부여되는 점수
- **비밀_공유**: 앉아있는 사용자가 선택된 서있는 사용자들과 비공개로 좌석 정보를 공유할 수 있는 기능
- **호차_위치**: 열차 내 특정 지하철 칸의 위치 (앞쪽/중간/뒤쪽)
- **좌석_위치**: 지하철 칸 내 좌석의 대략적인 위치

## 요구사항

### 요구사항 1: 사용자 역할 선택

**사용자 스토리:** 지하철 승객으로서, 나의 현재 상태(앉아있음 또는 서있음)를 선택하고 싶다. 그래야 내 상황에 맞는 적절한 기능에 접근할 수 있다.

#### 수락 기준

1. 사용자가 애플리케이션을 열 때, 지하철_시스템은 "앉아있는 사용자"와 "서있는 사용자" 역할 선택 옵션을 표시해야 한다
2. 사용자가 "서있는 사용자"를 선택할 때, 지하철_시스템은 "현재 지하철 안" 및 "탑승 예정" 옵션을 제공해야 한다
3. 사용자가 역할을 선택할 때, 지하철_시스템은 해당 역할에 적합한 워크플로우로 이동해야 한다

### 요구사항 2: 지하철 노선 및 열차 선택

**사용자 스토리:** 사용자로서, 내가 탑승한 지하철 노선과 특정 열차를 선택하고 싶다. 그래야 시스템이 내 위치를 정확히 추적하고 관련 사용자들과 연결해줄 수 있다.

#### 수락 기준

1. 사용자가 위치를 선택해야 할 때, 지하철_시스템은 사용 가능한 모든 지하철 노선을 표시해야 한다
2. 사용자가 지하철 노선을 선택할 때, 지하철_시스템은 해당 노선의 완전한 노선도를 보여줘야 한다
3. 사용자가 현재 역을 선택할 때, 지하철_시스템은 상행 및 하행 열차 옵션을 제공해야 한다
4. 열차 옵션이 표시될 때, 지하철_시스템은 열차 식별 번호를 기반으로 한 수동 열차 선택을 허용해야 한다
5. 지하철_시스템은 사용자가 선택한 방향과 시간을 기반으로 자동 열차 추정을 제공해야 한다

### 요구사항 3: 앉아있는 사용자 정보 공유

**사용자 스토리:** 앉아있는 사용자로서, 내 좌석 정보와 하차역을 공유하고 싶다. 그래야 서있는 사용자들이 내가 떠날 때 내 좌석을 찾아서 차지할 수 있다.

#### 수락 기준

1. 앉아있는 사용자가 열차를 선택할 때, 지하철_시스템은 호차 위치 선택(앞쪽/중간/뒤쪽 또는 직접 입력)을 요청해야 한다
2. 호차 위치가 선택될 때, 지하철_시스템은 대략적인 좌석 선택을 위한 좌석 배치도 이미지를 표시해야 한다
3. 좌석 위치가 선택될 때, 지하철_시스템은 외모 설명 입력을 요청해야 한다
4. 외모 설명이 입력될 때, 지하철_시스템은 하차역 선택을 요청해야 한다
5. 모든 정보가 제공될 때, 지하철_시스템은 좌석 정보를 게시하고 +1 기여도 점수를 부여해야 한다
6. 지하철_시스템은 공개 공유 또는 선택된 사용자들과의 비공개 공유 옵션을 제공해야 한다

### 요구사항 4: 서있는 사용자 좌석 발견

**사용자 스토리:** 현재 지하철에 있는 서있는 사용자로서, 곧 내릴 사용자들의 사용 가능한 좌석을 보고 싶다. 그래야 그 좌석들을 차지하기 위해 이동할 수 있다.

#### 수락 기준

1. 서있는 사용자가 "현재 지하철 안"을 선택할 때, 지하철_시스템은 열차 선택 과정을 안내해야 한다
2. 열차 선택이 완료될 때, 지하철_시스템은 탑승 구간 입력을 요청해야 한다
3. 구간 정보가 제공될 때, 지하철_시스템은 곧 내릴 사용자들의 목록을 표시해야 한다
4. 좌석 목록이 표시될 때, 지하철_시스템은 각 좌석에 대해 호차 위치, 대략적인 좌석 위치, 외모 설명, 대기 중인 사용자 수를 보여줘야 한다
5. 서있는 사용자가 특정 좌석을 선택할 때, 지하철_시스템은 해당 사용자가 하차할 때의 알림 설정을 제공해야 한다
6. 지하철_시스템은 사용자가 대기를 취소하거나 다른 좌석을 선택할 수 있도록 허용해야 한다

### 요구사항 5: 사전 탑승 계획

**사용자 스토리:** 지하철 탑승을 계획하는 사용자로서, 어떤 열차에 사용 가능한 좌석이 있는지 보고 싶다. 그래야 탑승할 최적의 열차를 선택할 수 있다.

#### 수락 기준

1. 사용자가 "탑승 예정"을 선택할 때, 지하철_시스템은 출발역과 도착역 선택을 요청해야 한다
2. 역들이 선택될 때, 지하철_시스템은 지하철 노선 선택을 요청해야 한다
3. 노선이 선택될 때, 지하철_시스템은 열차 도착 알림을 제공해야 한다
4. 지하철_시스템은 곧 내릴 사용자 수가 많은 열차를 우선적으로 추천해야 한다
5. 사용자가 열차에 탑승할 때, 지하철_시스템은 자동으로 "현재 지하철 안" 모드로 전환해야 한다

### 요구사항 6: 알림 및 경고 시스템

**사용자 스토리:** 좌석을 기다리는 서있는 사용자로서, 내가 원하는 좌석이 사용 가능해질 때 적시에 알림을 받고 싶다. 그래야 빠르게 이동해서 그 좌석을 차지할 수 있다.

#### 수락 기준

1. 앉아있는 사용자가 하차역으로부터 1-2정거장 떨어져 있을 때, 지하철_시스템은 대기 중인 사용자들에게 자동 알림을 보내야 한다
2. 같은 좌석을 여러 사용자가 기다리고 있을 때, 지하철_시스템은 모든 대기 사용자들에게 동시에 알림을 보내야 한다
3. 앉아있는 사용자가 열차에서 내릴 때, 지하철_시스템은 대기 사용자들에게 좌석 획득 성공 또는 실패 확인을 요청해야 한다
4. 지하철_시스템은 앉아있는 사용자가 다른 사람들이 자신의 좌석을 기다리고 있을 때 알림을 받을 수 있도록 허용해야 한다

### 요구사항 7: 기여도 점수 관리

**사용자 스토리:** 좌석 공유 생태계에 참여하는 사용자로서, 기여도 점수를 획득하고 사용하고 싶다. 그래야 프리미엄 기능에 접근하고 다른 사용자들에게 우선순위를 받을 수 있다.

#### 수락 기준

1. 앉아있는 사용자가 좌석 정보를 게시할 때, 지하철_시스템은 +1 기여도 점수를 부여해야 한다
2. 서있는 사용자가 요청을 게시할 때, 지하철_시스템은 +1 기여도 점수를 부여해야 한다
3. 다른 사용자가 공유된 정보로 인해 성공적으로 좌석을 차지할 때, 지하철_시스템은 앉아있던 사용자에게 +3 추가 기여도 점수를 부여해야 한다
4. 사용자가 높은 기여도 점수를 가질 때, 지하철_시스템은 비공개 좌석 공유 선택에서 그들을 우선순위로 두어야 한다
5. 지하철_시스템은 사용자 기여도 점수와 레벨을 표시해야 한다

### 요구사항 8: 비밀 공유 기능

**사용자 스토리:** 앉아있는 사용자로서, 사용자들의 사연이나 기여도 점수를 기반으로 특정 사용자들과 선택적으로 내 좌석 정보를 공유하고 싶다. 그래야 좌석이 가장 필요한 사용자들을 도울 수 있다.

#### 수락 기준

1. 좌석 정보를 게시할 때, 지하철_시스템은 공개 또는 비공개 공유 옵션을 제공해야 한다
2. 비공개 공유가 선택될 때, 지하철_시스템은 대기 사용자들의 사연과 기여도 점수를 표시해야 한다
3. 대기 사용자들을 검토할 때, 지하철_시스템은 앉아있는 사용자가 좌석 정보의 특정 수신자들을 선택할 수 있도록 허용해야 한다
4. 지하철_시스템은 선택된 사용자들에게만 비공개 알림을 보내야 한다
5. 비공개 공유를 위해 선택된 사용자가 없을 때, 지하철_시스템은 좌석 정보를 비공개로 유지해야 한다

### 요구사항 9: 사용자 사연 및 상황 공유

**사용자 스토리:** 서있는 사용자로서, 좌석이 필요한 이유나 사연을 공유하고 싶다. 그래야 앉아있는 사용자들이 좌석 정보를 비공개로 공유할지 결정할 때 내 상황을 고려할 수 있다.

#### 수락 기준

1. 서있는 사용자가 좌석 대기를 설정할 때, 지하철_시스템은 선택적 사연 입력 필드를 제공해야 한다
2. 사연이 제공될 때, 지하철_시스템은 비공개 공유 선택 중에 앉아있는 사용자들에게 그것을 표시해야 한다
3. 지하철_시스템은 가독성을 유지하기 위해 사연 길이를 제한해야 한다
4. 지하철_시스템은 사용자가 자신의 사연을 업데이트하거나 제거할 수 있도록 허용해야 한다

### 요구사항 10: 좌석 획득 피드백 시스템

**사용자 스토리:** 좌석 차지를 시도한 사용자로서, 좌석 획득의 성공 또는 실패에 대한 피드백을 제공하고 싶다. 그래야 시스템이 정확성을 유지하고 적절한 점수를 부여할 수 있다.

#### 수락 기준

1. 앉아있던 사용자가 하차하고 열차가 이동할 때, 지하철_시스템은 대기 사용자들에게 좌석 획득 확인을 요청해야 한다
2. 좌석 획득이 성공적일 때, 지하철_시스템은 선택적 리뷰 및 평점을 요청해야 한다
3. 좌석 획득이 실패할 때, 지하철_시스템은 사용자의 +1 기여도 점수를 유지하고 좌석 발견으로 돌아가야 한다
4. 지하철_시스템은 사용자 제어를 위해 "내렸어요" 및 "대기 취소" 버튼을 제공해야 한다
5. 획득 피드백이 제공될 때, 지하철_시스템은 그에 따라 기여도 점수를 업데이트해야 한다
